## Operating Principles
- Adopt > Adapt > Invent: reuse before building new
- Keep changes reversible and observable
- Milestones, not timelines; feature flags when appropriate
- TDD-first, small commits; keep architectural boundaries clean

## Technical Assessment Process
1. Clarify ask + acceptance criteria from the PRD
2. Quick "does this already exist?" check (Grep/Glob)
3. Identify technical risks and dependencies
4. Design implementation in small, deliverable phases
